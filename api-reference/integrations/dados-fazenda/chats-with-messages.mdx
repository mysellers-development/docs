---
title: 'Exportar conversas com mensagens'
openapi: 'GET /integrations/dados-fazenda/chats-with-messages'
---

<Tip>
  **Caso de uso:** voce quer puxar o historico de conversas de um workspace para analisar interacoes — separando o que foi cliente, atendente humano ou IA, com recorte por periodo. Este endpoint devolve a lista de chats paginada, com as mensagens ja embutidas e filtradas no servidor.
</Tip>

## Como funciona

Para cada chat do workspace cujo `lastMessageAt` esta dentro da janela `from` / `to`:

1. Trazemos o chat e seus dados basicos (lead, telefone, status, data de inicio).
2. Embutimos as mensagens do chat ja filtradas pela mesma janela e pelos filtros opcionais (`senderType`, `direction`).
3. Chats que ficariam sem mensagens depois dos filtros sao **excluidos da resposta** — voce nao recebe item vazio.

A resposta e paginada por **cursor** — voce nunca recebe tudo num request so. Cada pagina vem ordenada por `lastMessageAt DESC` (chats mais recentes primeiro).

## Autenticacao

<Note>
  Requer `x-api-key` vinculada a um workspace. O `workspaceId` e inferido automaticamente do token. Requests sem API Key de workspace recebem `403 Forbidden`.
</Note>

## Janela de tempo

- **Default:** ultimos 30 dias (se nao mandar `from`/`to`).
- **Maxima:** 365 dias. Acima disso retorna `400 Bad Request` — divida em janelas menores.
- **Formato:** ISO 8601 com timezone (ex: `2026-04-07T00:00:00Z`).
- A janela e aplicada em `lastMessageAt` do chat **e** em `createdAt` da mensagem. Um chat antigo sem mensagens novas no periodo nao aparece. Mensagens fora do periodo sao filtradas mesmo que o chat esteja dentro.

## Filtros opcionais

| Filtro | Valores | Comportamento |
|---|---|---|
| `senderType` | csv: `lead`, `agent`, `ai`, `system` | So traz mensagens cujo `senderType` esta na lista. Ex: `?senderType=ai,agent` traz so respostas (sem mensagens do lead). |
| `direction` | `inbound` ou `outbound` | `inbound` = do lead pra empresa. `outbound` = da empresa pro lead (atendente ou IA). |

## Como identificar IA × humano

Cada mensagem traz dois campos pra isso. Use o que for mais conveniente:

- **`senderType`** (granular) — `lead` | `agent` | `ai` | `system`
- **`authorRole`** (simplificado) — `human` | `ai` | `system`
  - `lead` e `agent` colapsam em `"human"`
  - `ai` permanece `"ai"`
  - `system` permanece `"system"` (notificacoes automaticas, follow-ups disparados pelo sistema)

`authorRole` e o atalho pra "foi IA ou humano". `senderType` e util quando voce precisa distinguir cliente final (`lead`) de atendente humano (`agent`).

## Paginacao por cursor

Fluxo padrao:

1. Primeiro request **sem** `cursor`.
2. Se `meta.hasMore === true`, faca o proximo request usando `cursor=meta.nextCursor`.
3. Repita ate `meta.hasMore === false` (ou `meta.nextCursor === null`).

O `cursor` e o `id` do ultimo chat da pagina anterior. Mantenha os mesmos `from`, `to` e filtros ao paginar — alterar entre paginas pode pular chats.

## Limites

- **`limit`:** default `50`, maximo `500`. Acima disso e clampado.
- **Janela:** maxima 365 dias.
- **Volume estimado por pagina:** com `limit=500`, tipicamente ~1-3 MB de JSON. Chats outliers com muitas mensagens podem aumentar isso.

## Erros

| HTTP | Quando |
|---|---|
| `400` | `from > to`, janela > 365d, `senderType` ou `direction` invalido, `cursor` nao e UUID, `limit` fora de [1, 500] |
| `401` | Sem `x-api-key` ou key invalida |
| `403` | API Key nao vinculada a um workspace |

## Exemplos

### Ultimos 30 dias (default)

```bash
curl https://apiap.mysellers.com.br/integrations/dados-fazenda/chats-with-messages \
  -H "x-api-key: ms_live_..." \
  -G --data-urlencode "limit=50"
```

### 90 dias, so respostas (IA + humano), sem mensagens do lead

```bash
curl https://apiap.mysellers.com.br/integrations/dados-fazenda/chats-with-messages \
  -H "x-api-key: ms_live_..." \
  -G \
  --data-urlencode "from=2026-02-07T00:00:00Z" \
  --data-urlencode "to=2026-05-07T23:59:59Z" \
  --data-urlencode "senderType=ai,agent" \
  --data-urlencode "limit=500"
```

### Paginar a proxima pagina

```bash
curl https://apiap.mysellers.com.br/integrations/dados-fazenda/chats-with-messages \
  -H "x-api-key: ms_live_..." \
  -G \
  --data-urlencode "from=2026-02-07T00:00:00Z" \
  --data-urlencode "to=2026-05-07T23:59:59Z" \
  --data-urlencode "cursor=d244347d-6e7c-486d-819d-5819a76a58ba" \
  --data-urlencode "limit=500"
```
