## Base URL

```
https://apiap.mysellers.com.br
```

## Autenticacao

A API do MySellers suporta dois metodos de autenticacao:

<CardGroup cols={2}>
  <Card title="JWT Bearer Token" icon="key">
    Obtido via `POST /auth/login` com email e senha. Ideal para aplicacoes que gerenciam sessoes de usuario.
    ```bash
    Authorization: Bearer eyJhbGciOiJI...
    ```
  </Card>
  <Card title="API Key" icon="lock">
    Gerada em **Configuracoes > Tokens** no painel do MySellers. Vinculada automaticamente a um workspace — nao precisa enviar `workspaceId` nos requests.
    ```bash
    x-api-key: ms_live_abc123...
    ```
  </Card>
</CardGroup>

<Tip>
  Recomendamos usar **API Key** para integracoes server-to-server. O `workspaceId` e preenchido automaticamente pelo token, simplificando os requests.
</Tip>

## Rate Limiting

A API possui tres niveis de rate limiting:

| Nivel | Limite | Janela |
|-------|--------|--------|
| Short | 10 requests | 1 segundo |
| Medium | 50 requests | 10 segundos |
| Long | 200 requests | 60 segundos |

Quando o limite e atingido, a API retorna `429 Too Many Requests`.

## Formato de Resposta

Todas as respostas sao em JSON. Endpoints de listagem retornam dados paginados:

```json
{
  "data": [...],
  "total": 150,
  "page": 1,
  "limit": 20
}
```

## Erros

Erros seguem o formato padrao:

```json
{
  "statusCode": 400,
  "message": "fullName must be longer than or equal to 2 characters",
  "error": "Bad Request"
}
```

| Codigo | Significado |
|--------|-------------|
| 400 | Dados invalidos no request |
| 401 | Token ausente ou expirado |
| 403 | Sem permissao para acessar o recurso |
| 404 | Recurso nao encontrado |
| 429 | Rate limit atingido |
| 500 | Erro interno do servidor |
